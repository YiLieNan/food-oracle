# 🥢 食神签 — 野生食神的日常审判

一个纯娱乐向的应用：让「食神」帮你决定今天吃什么，附带毒舌点评。
支持 **Web 端 + 手机 App (Android)**，后端可部署到云端 24h 运行。

## 📂 项目结构

```
食神签/
├── server.js              # 后端服务器 (Express + SQLite)
├── 食神签.html            # Web 前端（桌面浏览器）
├── mobile/index.html      # 手机端前端（Capacitor 封装入口）
├── capacitor.config.ts    # Capacitor 配置
├── android/               # Android 原生项目
├── Dockerfile             # Docker 镜像
├── docker-compose.yml     # Docker Compose 一键部署
├── railway.toml           # Railway 部署配置
└── package.json
```

## ☁️ 部署到 Railway（照着点就行，3 分钟）

### 第 1 步：打开 Railway

浏览器打开 https://railway.app

点右上角 **Login** → 选 **GitHub** → 授权登录

### 第 2 步：创建项目

登录后会看到 Dashboard 页面。点那个大大的紫色按钮：

> **+ New Project**

### 第 3 步：选择仓库

选 **Deploy from GitHub repo** → 会列出你的仓库

找到 **YiLieNan/food-oracle** → 点击它

> 如果没看到仓库，点 "Configure Railway on GitHub" 授权访问

### 第 4 步：添加存储卷（重要！数据库不能丢）

项目创建后会自动开始部署，先让它跑完（约 2 分钟）。

部署完成后，在项目页面：

1. 点顶部 **Settings** 标签
2. 左侧找到 **Volumes** → 点 **+ Add Volume**
3. 这样填：
   - **Mount path**：`/app/data`
   - **Size**：`1 GB`
4. 点 **Create**

### 第 5 步：（可选）开启 AI 神谕

同一 Settings 页面，找到 **Environment Variables**（或 **Variables**）：

点 **+ New Variable**，添加：

| Name | Value |
|------|-------|
| `AI_API_KEY` | 你的 Groq API Key |

> 免费获取 Key：打开 https://console.groq.com/keys，注册即送

### 第 6 步：重新部署

点右上角 **Deploy** → **Redeploy**（让 Volume 和变量生效）

### 第 7 步：获取你的云端地址

点 **Settings** → 往下翻到 **Domains**

找到 `Generate Domain`，生成后会得到一个类似这样的地址：

```
food-oracle.up.railway.app
```

这个就是你的服务器地址！✅

### 第 8 步：填到手机 App 里

打开食神签 App → 点右上角 ⚙️ → 在「服务器地址」填：

```
https://food-oracle.up.railway.app
```

（替换成你实际拿到的域名）

填完后自动检测，显示「✅ 已连接」就搞定了！🎉

### 方案 B：自己有 VPS（阿里云/腾讯云等）

```bash
# 1. SSH 登录 VPS，安装 Docker
curl -fsSL https://get.docker.com | sh

# 2. 克隆项目
git clone <你的仓库地址> && cd 食神签

# 3. 一键启动（加 AI 神谕）
AI_API_KEY=你的key docker compose up -d

# 或不启用 AI：
docker compose up -d

# 4. 查看状态
docker compose logs -f

# 5. 确保防火墙开放 3456 端口
# 阿里云/腾讯云需在安全组中放行 TCP 3456
```

### 方案 C：直接运行（已有 Node.js 的 VPS）

```bash
npm install
PORT=3456 AI_API_KEY=你的key npm start
# 用 pm2 保持后台运行：
npx pm2 start server.js --name food-oracle
```

> **注意**：无论哪种方案，部署后记下服务器 URL（如 `http://你的IP:3456` 或 `https://域名`），手机 App 需要填写。

## 📱 手机 App 构建

### 前置条件
- Android Studio + JDK 17+
- 已部署后端到云端（或本地 PC 运行中）

### 构建 APK

```bash
npm install
npm run mobile:sync        # 同步 web 资源
npm run mobile:open        # 在 Android Studio 中打开
# 然后：Build → Build APK
```

### 使用流程
1. 安装 APK 到手机
2. 首次启动自动弹出设置 → 填入云端服务器地址
3. 摇晃手机或点击签筒 → 求签 🔮

## 🔧 命令参考

| 命令 | 说明 |
|------|------|
| `npm start` | 启动后端（本地） |
| `npm run mobile:sync` | 同步资源到 Android 项目 |
| `npm run mobile:open` | 打开 Android Studio |
| `docker compose up -d` | Docker 部署到云端 |
| `docker compose logs -f` | 查看服务器日志 |

## ⚙️ 环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `PORT` | 服务端口 | `3456` |
| `DB_PATH` | SQLite 数据库路径 | `./food_oracle.db` |
| `AI_PROVIDER` | AI 提供商：groq/gemini/siliconflow/deepseek | `groq` |
| `AI_API_KEY` | AI API 密钥 | (空，不启用) |
| `AI_MODEL` | AI 模型名 | 各提供商默认值 |
